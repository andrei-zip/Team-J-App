Testing Report - Team-J-App


This page (test.html) will describe our teamwork and efforts approach for our Web Application

Testing Methods

To ensure the correctness of our Web App, we applied a structured testing approach, which included the following:

  • Unit Testing using console logs and API response validation
  • Manual testing for user interaction and UI/UX validation
  • User Acceptance Testing (UAT) with real users

Unit Testing

During development, we implemented unit tests to verify the correctness of key functions:

  • Ensuring fetchQuestion() retrieves and displays questions correctly
  • Validating submitAnswer() processes responses and updates scores properly
  • Testing updateLocation() to ensure it correctly interacts with the API
  • Confirming skipQuestion reduces score and loads the next question

API testing

We validated API responses using browser DevTools. The main tests included:

  • Checking if the /start endpoint correctly initialises a session
  • Validating responses from the /question endpoint
  • Ensuring /answer updates the score and feedback messages correctly
  • Testing the /leaderboard API call to fetch and display results
  • And finally, being sure that The Game, overall, is playable

User Acceptance Test (UAT)

We asked real-users to testing our Treasure Hunt to validate the application's usability! The main feedback from users:

  • The UI is intuitive and easy to navigate
  • The leaderboard correctly updates after completing the treasure hunt
  • Users suggested a confirmation prompt before skipping a question

Issues Identified and Fixed

Here are some key issues we found and fixed:

Issue Status Solution
Score went negative after multiple incorrect answers
Leaderboard did not display correctly Fixed Updated API call parameters
Location updates were not being registered properly Fixed Ensured correct API call structure

Future Improvements
